# Spring Cloud Alibaba

组件

  • Nacos：阿里巴巴开源产品，一个更易于构建云原生应用的动态服务发现,配置管理和服务管理平台。
  • Sentinel：阿里巴巴开源产品，把流量作为切入点,从流量控制,熔断降级,系统负载保护等多个维度保护服务的稳定性。
  • RocketMQ：Apache RocketMQ 是一款基于Java 的高性能、高吞吐量的分布式消息和流计算平台。
  • Dubbo：Apache Dubbo 是一款高性能的 Java RPC 框架。
  • Seata：阿里巴巴开源产品，一个易于使用的高性能微服务分布式事务解决方案。
  • Alibaba Cloud OSS：阿里云对象存储服务器（Object Storage Service，简称OSS），是阿里云提供的海量、安全、低成本、高可靠的云存储服务。
  • Alibaba Cloud Schedulerx：阿里中间件团队开发的一款分布式调度产品,支持周期性的任务与固定时间点触发任务。

通过 Spring Cloud Alibaba 的这些组件，我们只需要添加一些注解和少量配置，就可以将 Spring Cloud 应用接入阿里微服务解决方案，通过阿里中间件来迅速搭建分布式应用系统。

public interface UserFeignService {
    
    @GetMapping("user/query/{name}")
    TestUser queryByName(@PathVariable("name") String name);
    
    @GetMapping("user/query")
    TestUser queryUserByName(@RequestParam("name") String name);
    
    @PostMapping("user/add")
    TestUser addUser(@RequestBody TestUser user);
}

@Data
public class TestUser implements Serializable {
    
    private String name;
    
    private Integer age;
}


@RefreshScope
@RestController
public class TestController {
    
    @Value("${config.info}")
    private String configInfo;
    
    @Resource
    private UserFeignService userFeignService;
    
    @GetMapping("test")
    public String test() {
        return configInfo;
    }
    
    
    @GetMapping("query/{name}")
    public TestUser queryByName(@PathVariable String name) {
        return userFeignService.queryUserByName(name);
    }
    
    @GetMapping("query")
    public TestUser queryUserByName(@RequestParam String name) {
        return userFeignService.queryUserByName(name);
    }
    
    @PostMapping("add")
    public TestUser addUser(@RequestBody TestUser user) {
        return userFeignService.addUser(user);
    }
}

# Sentinel

Sentinel 作为熔断降级组件是替代hystrix的,这里还是照例只演示使用

Sentinel提供了一个控制台来动态的为一些资源设置降级熔断逻辑,下载控制台 https://github.com/alibaba/Sentinel/releases

这里使用的是 1.8.2版本,java -jar 启动就行 默认控制台端口号是 8080 http://localhost:8080/ 用户名密码 sentinel

这里提供多种方式使用

  • 适配主流框架自动定义资源
  • 通过 SphU 手动定义资源
  • 通过 SphO 手动定义资源
  • 注解方式定义资源

演示 代码写完之后,还要在控制台配置规则才行,比如限流或者熔断规则


@Slf4j
@RestController
public class SentinelController {
    
    /**
     * 通过 SphU 手动定义资源
     *
     * @return
     */
    @GetMapping("testSphU")
    public String testSphU() {
        Entry entry = null;
        try {
            entry = SphU.entry("testSphU");
            //您的业务逻辑 - 开始
            log.info("服务访问成功------testSphU");
            return "服务访问成功------testSphU";
            //您的业务逻辑 - 结束
        } catch (BlockException e1) {
            //流控逻辑处理 - 开始
            log.info("服务被限流------testSphU");
            return "服务被限流------testSphU";
            //流控逻辑处理 - 结束
        } finally {
            if (entry != null) {
                entry.exit();
            }
        }
    }
    
    @GetMapping("testSphO")
    public String testSphO() {
        if (SphO.entry("testSphO")) {
            // 务必保证finally会被执行
            try {
                log.info("服务访问成功------testSphO");
                return "服务访问成功------testSphO";
            } finally {
                SphO.exit();
            }
        } else {
            // 资源访问阻止，被限流或被降级
            //流控逻辑处理 - 开始
            log.info("testSphO 服务被限流");
            return "testSphO 服务被限流";
            //流控逻辑处理 - 结束
        }
    }
    
    /**
     * 通过 Sentinel 控制台定义流控规则
     */
    @GetMapping("/testResource")
    @SentinelResource(value = "testResource", blockHandler = "blockHandlerTestD") //通过注解定义资源
    public String testResource() {
        log.info("服务访问成功------testResource");
        return "服务访问成功------testResource";
    }
    
    /**
     * 限流之后的逻辑
     *
     * @param exception
     * @return
     */
    public String blockHandlerTestD(BlockException exception) {
        log.info(Thread.currentThread().getName() + "testResource服务访问失败! 您已被限流，请稍后重试");
        return "testResource服务访问失败! 您已被限流，请稍后重试";
    }
}
