# Java代理

# 静态代理

所谓静态代理就是在编译器,写好代码,通过编码注入的方式实现

public interface IUserService {
    
    String userId();

}

public interface IProxyService {
    
    void before();
    
    void after();
}

public class UserProxyService implements IProxyService {
    
    @Override
    public void before() {
        System.out.println("开始");
    }
    
    @Override
    public void after() {
        System.out.println("后置");
    }
}

@AllArgsConstructor
public class UserService implements IUserService {
    
    private IProxyService proxyService;
    
    @Override
    public String userId() {
        proxyService.before();
        String id = "123";
        System.out.println("userService执行");
        proxyService.after();
        return id;
    }
}

public class Main {
    
    public static void main(String[] args) {
        IProxyService proxyService = new UserProxyService();
        IUserService userService = new UserService(proxyService);
        userService.userId();
    }
}

# 动态代理

动态代理是在运行期生成的字节码,编译器对代理后的代码是不可见的, 不过也是基于java多态的原理来实现,继承类或者实现接口,请参照静态代理对号入座,不论是我们自己写还是动态生成,结构一样

# JDK代理

2个重点

  • Proxy.newProxyInstance

创建基于接口的代理对象,3个参数

ClassLoader loader,

Class<?>[] interfaces,

InvocationHandler h

  • InvocationHandler

对象方法反射执行,在反射调用过程中对方法进行代理,也可以指定哪些方法需要代理,哪些不需要

案例,看代码

public interface IUserService {
    
    String userId();
}


public class UserService implements IUserService {
    
    @Override
    public String userId() {
        System.out.println("userService userId执行");
        return "123";
    }
}


public class JdkProxyFactory {
    
    public static IUserService getProxyService(IUserService target) {
        return (IUserService) Proxy.newProxyInstance(target.getClass().getClassLoader(),
                target.getClass().getInterfaces(), new UserServiceUserIdHandler(target));
    }
    
    private static class UserServiceUserIdHandler implements InvocationHandler {
        
        private IUserService target;
        
        UserServiceUserIdHandler(IUserService target) {
            this.target = target;
        }
        
        // method底层的方法无参数时，args为空或者长度为0
        @Override
        public Object invoke(Object proxy, Method method, @Nullable Object[] args) throws Throwable {
            String name = method.getName();
            Object result = null;
            if ("userId".equals(name)) {
                // 扩展前置
                System.out.println("前置处理");
                // 访问基础对象
                try {
                    result = method.invoke(target, args);
                } catch (Exception e) {
                    // 扩展异常处理
                    System.err.println("IUserService 出异常了" + e);
                }
                // 扩展后置
                System.out.println("后置处理");
            } else {
                result = method.invoke(target, args);
            }
            
            return result;
        }
    }
}


public class Main {
    
    public static void main(String[] args) {
        IUserService target = new UserService();
        IUserService userService = JdkProxyFactory.getProxyService(target);
        userService.userId();
        userService.hello();
    }
}

至于为什么Jdk代理只能代理接口,不能代理类,主要还是生成的代理类是 extends Proxy 由于java单继承原则,只能通过 implements 来扩展了

# cglib代理

cglib 是基于ASM 字节码增强技术实现了,可以对类进行代理

2个关键类

  • MethodInterceptor
  • Enhancer

案例:

public class UserService {
    
    public String userId() {
        System.out.println("userService userId执行");
        return "123";
    }
    
    public void hello() {
        System.out.println("hello");
    }
}

public class UserSreviceMyMethodInterceptor implements MethodInterceptor {
    
    @Override
    public Object intercept(Object o, Method method, Object[] objects, MethodProxy methodProxy) throws Throwable {
        String name = method.getName();
        Object result = null;
        switch (name) {
            case "userId":
                System.out.println("cglib增强--前置");
                try {
                    result = methodProxy.invokeSuper(o, objects);
                } catch (Exception e) {
                    System.out.println("cglib增强--异常");
                }
                System.out.println("cglib增强--后置");
                break;
            default:
                result = methodProxy.invokeSuper(o, objects);
                break;
        }
        return result;
    }
}

public class Main {
    
    public static void main(String[] args) {
        Enhancer enhancer = new Enhancer();
        enhancer.setSuperclass(UserService.class);
        enhancer.setCallback(new UserSreviceMyMethodInterceptor());
        
        UserService userService = (UserService) enhancer.create();
        userService.userId();
        userService.hello();
    }
}
