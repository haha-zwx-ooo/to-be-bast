# Future和CompletableFuture

在 多线程(一) 和 多线程(二) 章节中,我们了解什么是线程,以及如何创建,启动和停止线程,但是 无论是extend Thread 还是 implements Runnable 都无法获取返回值和异常信息,于是就出现了今天的主题

  • Future<V>
  • CompletableFuture<T>
public class CallAbleDemo {
    
    public static void main(String[] args) throws ExecutionException, InterruptedException {
        ThreadCall call = new ThreadCall();
        FutureTask<String> futureTask = new FutureTask<>(new ThreadCall());
        Thread t1 = new Thread(futureTask, "t1");
        //正常启动
        t1.start();
        //尝试获取返回结果
        System.out.println("****** result=" + futureTask.get());
    }
}

class ThreadCall implements Callable<String> {
    
    @Override
    public String call() throws Exception {
        TimeUnit.MILLISECONDS.sleep(500);
        return "ok";
    }
}

public class CompletableFutureDemo {
    
    public static void main(String[] args) {
        CompletableFuture<String> future1 = CompletableFuture.supplyAsync(() -> {
            try {
                TimeUnit.MILLISECONDS.sleep(100);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            return "ok";
        });
        future1.thenAccept(resp -> {
            System.out.println("future1执行成功结果是: " + resp);
            System.out.println("继续执行");
        });
        
        System.out.println("do something else");
        future1.join();
    }
}

# 区别

在Java中，Future和CompletableFuture都是用于异步编程的接口/类，它们都可以用于获取异步任务的结果。但是它们之间有一些不同之处，包括以下几点：

  1. Future是在Java 5中引入的，而CompletableFuture是在Java 8中引入的。因此，CompletableFuture提供了更多的异步编程功能和方法。

  2. Future是一个接口，表示一个异步计算的结果。它提供了一个get()方法来获取结果。但是，如果异步计算尚未完成，get() 方法将会阻塞，直到结果可用或者抛出异常。如果你希望取消一个任务，你需要调用cancel()方法。但是，如果任务已经开始执行，你不能取消它。

  3. CompletableFuture是一个实现了Future接口的类。它提供了与Future相同的get() 方法来获取结果，但是还提供了一些其他方法，如join() 和getNow()等。它还提供了一些非常方便的方法，如thenApply()和thenAccept() 等，用于在任务完成后执行一些操作。与Future不同的是，你可以使用complete()方法手动完成任务，也可以使用cancel()方法取消任务。

  4. CompletableFuture支持更高级的操作，例如组合多个异步任务、超时处理、异常处理等。

总之，Future和CompletableFuture都是用于获取异步任务的结果，但是CompletableFuture提供了更多的异步编程功能和方法，能够更好地满足异步编程的需求。
