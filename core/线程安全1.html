# 线程安全

# 什么是线程安全

线程安全是指在多线程环境中，多个线程访问同一个共享资源时，不会产生不正确的结果。在线程安全的情况下，多线程访问共享资源也不会发生死锁、饥饿等问题。

一个线程安全的程序在多线程环境下，能够正确地完成操作。换句话说，线程安全是指多个线程对同一份资源进行操作时，不会出现意外的结果。线程安全的程序必须确保在多线程环境中，对于共享的数据或资源，不会产生数据的冲突和不一致的问题。

# 举例

public class NotSafeDemo {
    
    
    private int index = 0;
    
    public static void main(String[] args) {
        NotSafeDemo demo = new NotSafeDemo();
        Thread t1 = new Thread(() -> {
            for (int i = 0; i < 10; i++) {
                demo.addIndex();
            }
        });
        Thread t2 = new Thread(() -> {
            for (int i = 0; i < 10; i++) {
                demo.addIndex();
            }
        });
        t1.start();
        t2.start();
    }
    
    public void addIndex() {
        index++;
        System.out.println(Thread.currentThread().getName() + " -- " + index);
    }
}

# 如何解决线程安全问题

  1. synchronized关键字：可以用来修饰方法或代码块，使得同一时间只有一个线程能够进入临界区，从而保证线程安全。

  2. ReentrantLock类：也可以用来保证同一时间只有一个线程能够进入临界区。

  3. AtomicInteger类：可以用来保证对于某个变量的自增或自减操作是原子性的，从而保证线程安全。

  4. ConcurrentHashMap类：是一种线程安全的哈希表，可以在并发访问时提供高效的性能。

  5. CopyOnWriteArrayList类：是一种线程安全的列表，可以在并发访问时提供高效的性能。

  6. volatile关键字：可以用来保证某个变量在多个线程之间的可见性，从而保证线程安全。

  7. ThreadLocal类：可以用来实现线程本地存储，从而保证线程安全。
