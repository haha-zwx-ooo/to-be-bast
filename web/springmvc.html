# springMvc

上面讲了servlet,了解到 在web容器中,需要构建一个 servlet 来处理http请求,那么springmvc 是怎么做的呢

DispatcherServlet 是springmvc中处理 http请求的核心类,相当于之前servlet我们写的HelloServlet,springmvc 把他加入到 web容器中,同时设置 servletMapping (/*) 这样 DispatcherServlet就可以处理所有的请求,具体流程如下:

  1. 扫描@Controller 或者 @RestController 并扫描 @RequestMapping 注解和方法, 初始化这些bean,并将url映射关系保存在 AbstractUrlHandlerMapping handlerMap 中
  2. 根据DispatcherServlet 中的HttpServletRequest 获取url,根据url获取controller bean
  3. 获取到的controller 反射获取 @RequestParam 参数 并执行对应url方法,封装返回值
