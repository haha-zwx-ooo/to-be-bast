# 接口

在Java中，接口（Interface）是一种特殊的抽象类，它定义了一组公共的方法和常量，但没有具体的实现。接口用于定义类的行为，相当于定义了一个合约或契约，规定了类必须实现的方法和属性，但是具体的实现由实现接口的类来完成。

Java接口通常用于实现多继承，一个类可以实现多个接口。接口的定义使用interface关键字，具体语法如下：

public interface InterfaceName {
    
    // 常量定义
    public static final int CONSTANT_VALUE = 100;
    
    // 方法定义
    public void method1();
    
    public String method2(int parameter);
}

上面的代码定义了一个接口InterfaceName，其中包含一个常量CONSTANT_VALUE和两个方法method1和method2。注意，在接口中定义的常量和方法都是public和static的，因此可以直接通过接口名称访问常量和方法，而不需要创建接口的实例。

要实现一个接口，可以使用implements关键字，在类的声明中声明该类实现哪些接口，然后在类中实现接口定义的所有方法，例如：


public class MyClass implements InterfaceName {
    
    public void method1() {
        // 实现method1方法
    }
    
    public String method2(int parameter) {
        // 实现method2方法
        return null;
    }
}

上面的代码中，MyClass实现了接口InterfaceName，并实现了接口中定义的两个方法method1和method2。

接口在Java中有很重要的作用，它可以使代码更加灵活、可扩展和可维护。通过实现接口，可以让代码具有更好的可读性和可复用性。

java8 接口新增了默认方法

public interface InterfaceDemo {
    
    int InterfaceDemo_i = 0;
    
    void say();
    
    default void say2() {
        System.out.println("hello");
    }

}
