# kafka

Kafka的核心是一个高吞吐量的、可水平扩展的、分布式的发布/订阅消息系统。它的设计目标是能够处理大规模的实时数据流，并提供高效的数据处理和传输机制。

以下是Kafka的一些核心特点：

  1. 消息持久化：Kafka使用持久性日志来存储消息，它不仅保证了消息的可靠性，而且能够支持消息的持久化和回放。

  2. 分布式架构：Kafka采用分布式架构，可以轻松扩展集群规模和负载均衡，并提供容错和高可用性支持。

  3. 高吞吐量：Kafka支持高吞吐量的数据传输和处理，能够处理数百万个消息的流量。

  4. 多样化的消费模型：Kafka支持多种消费模型，包括队列模型、发布/订阅模型和流处理模型等。

  5. 实时数据处理：Kafka的消息传递和处理机制能够实现实时的数据流处理，支持实时数据分析、监控和警报等应用场景。

总之，Kafka的核心是一个高吞吐量、可水平扩展的、分布式的消息传递和处理系统，它具有高可靠性、高效性和灵活性等特点，是处理大规模实时数据流的一个重要解决方案。Kafka的核心是一个高吞吐量的、可水平扩展的、分布式的发布/订阅消息系统。它的设计目标是能够处理大规模的实时数据流，并提供高效的数据处理和传输机制。

# kafka使用案例

使用java_sdk客户端发送消息到broker


<dependencies>
    <dependency>
        <artifactId>spring-kafka</artifactId>
        <groupId>org.springframework.kafka</groupId>
    </dependency>
</dependencies>

配置

server:
  port: 8080
spring:
  kafka:
    bootstrap-servers: 192.168.5.6:9092,192.168.5.7:9092
  profiles:
    active: consumer

---
spring:
  kafka:
    consumer:
      group-id: c1
      enable-auto-commit: false
      auto-offset-reset: earliest
    listener:
      ack-mode: manual_immediate
      poll-timeout: 500S

---
spring:
  kafka:
    consumer:
      group-id: c2
      enable-auto-commit: false
      auto-offset-reset: earliest
    listener:
      ack-mode: manual_immediate
      poll-timeout: 500S

---
spring:
  kafka:
    producer:
      acks: all



@Profile("consumer")
@Service
public class ConsumerService {
    
    /**
     * 监听kafka数据
     */
    @KafkaListener(topics = {"haha-test"})
    public void consumer(ConsumerRecord<?, ?> consumerRecord, Acknowledgment ack) {
        System.out.println("收到haha-test推送的数据" + consumerRecord.toString());
        ack.acknowledge();
    }
}

public class ConsumerService2 {
    
    /**

}

# kafka消息顺序消费

kafka默认是在同一个topic的不同分区进行负载均衡投递信息,同一个分区只能被一个消费者消费,这就可能导致消息投递顺序和消费顺序不一致问题

为了解决,投递消息时固定分区即可

# kafka消息高可靠

生产者 ack 消费者手动提交 禁止重试 涉及到 消费者扩缩容的时候的reBlance机制可能会导致重复消费,这个时候可以在消费者端消费提交后,在数据库写入消费的topic 分区 和对应的 offset, 在次启动的时候从数据库获取offset,并从当前offset开始读数据即可
