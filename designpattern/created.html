# 创建型设计模式

  • 简单工厂模式（Simple Factory）
    • 重要程度：4 （5为满分）
  • 工厂方法模式（Factory Method）
    • 重要程度：5
  • 抽象工厂模式（Abstract Factory）
    • 重要程度：5
  • 建造者模式（Builder）
    • 重要程度：2
  • 原型模式（Prototype）
    • 重要程度：3
  • 单例模式（Singleton）
    • 重要程度：4

本文不按照每个模式的内容讲解,而是按照代码演进的思路去覆盖这些设计模式:

# 案例

需求1: 设计一个计算器,让用户可以做加法运算

于是第一版本出现,完全是面向过程的写法.

public class Main {
    
    public static void demo1() {
        Scanner input = new Scanner(System.in);
        System.out.println("请输入数字");
        String n1 = input.nextLine();
        System.out.println("请输入数字");
        String n2 = input.nextLine();
        int resp = Integer.parseInt(n1) + Integer.parseInt(n2);
        System.out.println("结果是: " + resp);
    }
}

需求迭代了,1. 用户输入的可能是小数或者整数,第二 还要有减法运算 于是乎你想了一下,会不会以后有乘法,除法,还有聚合运算呢,于是你把运算符抽象成一个接口

public interface Operation {
    
    BigDecimal operation(BigDecimal x, BigDecimal y);
}

public class AddOperation implements Operation {
    
    @Override
    public BigDecimal operation(BigDecimal x, BigDecimal y) {
        return x.add(y);
    }
}

public class SubOperation implements Operation {
    
    @Override
    public BigDecimal operation(BigDecimal x, BigDecimal y) {
        return x.subtract(y);
    }
}

public class Main {
    
    public static void demo2() {
        Scanner input = new Scanner(System.in);
        System.out.println("请输入数字");
        BigDecimal b1 = input.nextBigDecimal();
        System.out.println("请输入数字");
        BigDecimal b2 = input.nextBigDecimal();
        System.out.println("请输入运算符");
        String opt = input.nextLine();
        Operation operation = null;
        switch (opt) {
            case "+":
                operation = new AddOperation();
                break;
            case "-":
                operation = new SubOperation();
                break;
            default:
                throw new IllegalArgumentException("非法运算符");
        }
        BigDecimal resp = operation.operation(b1, b2);
        System.out.println("结果是: " + resp);
    }
}

你感觉到 switch 那一大坨不太合适,但是没想好怎么处理他们,目的是根据用户输入产生不同对象, 好巧不巧你发现了,简单工厂设计模式,这不就是吗,于是你改了第三版

public class OperationFactory {
    
    public Operation createOperation(String type) {
        Operation operation = null;
        switch (type) {
            case "+":
                operation = new AddOperation();
                break;
            case "-":
                operation = new SubOperation();
                break;
            case "*":
            case "/":
            default:
                throw new IllegalArgumentException("非法运算符");
        }
        return operation;
    }

}

public class Main {
    
    public static void demo3() {
        Scanner input = new Scanner(System.in);
        System.out.println("请输入数字");
        BigDecimal b1 = input.nextBigDecimal();
        System.out.println("请输入数字");
        BigDecimal b2 = input.nextBigDecimal();
        System.out.println("请输入运算符");
        String opt = input.nextLine();
        OperationFactory factory = new OperationFactory();
        Operation operation = factory.createOperation(opt);
        BigDecimal resp = operation.operation(b1, b2);
        System.out.println("结果是: " + resp);
    }
}

需求又来了,老板说,我们这个计算器不能只是中文,还得给老外用,得有英文的,你想了想,又造出一个简单工厂模型, 这次你把输入输出封装成一个对象

public abstract class AbstractConsole {
    
    
    Scanner scanner;
    
    public AbstractConsole() {
        init();
    }
    
    public void init() {
        scanner = new Scanner(System.in);
    }
    
    public String inputString(String message) {
        System.out.println(message);
        String str = scanner.next();
        return str;
    }
    
    public BigDecimal inputNumber(String message) {
        System.out.println(message);
        BigDecimal bigDecimal = scanner.nextBigDecimal();
        return bigDecimal;
    }
    
    public void output(String message) {
        System.out.println(message);
    }
    
    public void destroy() {
        if (scanner != null) {
            scanner.close();
        }
    }
    
    public abstract String inputString();
    
    public abstract BigDecimal inputNumber();
    
    public abstract void output(BigDecimal result);
}

public class ChineseConsole extends AbstractConsole {
    
    @Override
    public String inputString() {
        return super.inputString("请输入操作符");
    }
    
    @Override
    public BigDecimal inputNumber() {
        return super.inputNumber("请输入数字");
    }
    
    @Override
    public void output(BigDecimal result) {
        super.output("结果是 :" + result);
    }
}

public class EnglishConsole extends AbstractConsole {
    
    @Override
    public String inputString() {
        return super.inputString("please input operation");
    }
    
    @Override
    public BigDecimal inputNumber() {
        return super.inputNumber("please input number");
    }
    
    @Override
    public void output(BigDecimal result) {
        super.output("result is :" + result);
    }
}

public class ConsoleFactory {
    
    AbstractConsole console;
    
    public AbstractConsole createConsole(String type) {
        switch (type) {
            case "china":
                console = new ChineseConsole();
                break;
            case "english":
                console = new EnglishConsole();
                break;
            default:
                throw new IllegalArgumentException("404");
        }
        return console;
    }
}

public class Main {
    
    public static void demo4() {
        ConsoleFactory consoleFactory = new ConsoleFactory();
        AbstractConsole console = consoleFactory.createConsole("china");
        String lang = console.inputString("请输入语言");
        console = consoleFactory.createConsole(lang);
        BigDecimal b1 = console.inputNumber();
        String opt = console.inputString();
        OperationFactory factory = new OperationFactory();
        Operation operation = factory.createOperation(opt);
        BigDecimal b2 = console.inputNumber();
        BigDecimal resp = operation.operation(b1, b2);
        console.output(resp);
    }
}

于是你越来越发现,面向接口编程的灵活性,于是你想Factory是具体实现类,如果factory变动怎么处理,要不要整成接口呢, 于是工厂方法模式开始了

public interface IOperationFactory {
    
    Operation createOperation(String type);
}

public class OperationFactory implements IOperationFactory {
    
    @Override
    public Operation createOperation(String type) {
        Operation operation = null;
        switch (type) {
            case "+":
                operation = new AddOperation();
                break;
            case "-":
                operation = new SubOperation();
                break;
            case "*":
            case "/":
                break;
            default:
                throw new IllegalArgumentException("非法运算符");
        }
        return operation;
    }

}

public interface IConsoleFactory {
    
    AbstractConsole createConsole(String type);
}

public class ConsoleFactory implements IConsoleFactory {
    
    AbstractConsole console;
    
    @Override
    public AbstractConsole createConsole(String type) {
        switch (type) {
            case "china":
                console = new ChineseConsole();
                break;
            case "english":
                console = new EnglishConsole();
                break;
            default:
                throw new IllegalArgumentException("404");
        }
        return console;
    }
}

但是你发现,以前只是很少代码,现在很多,以后变更了会更多,有没有必要说是每个点都搞一个工厂呢, 要不要弄成复合工厂,一个工厂把我所有的对象都生成了,于是抽象工厂来了

public interface ICalculatorFactory {
    
    Operation createOperation(String type);
    
    AbstractConsole createConsole(String type);
}


public class CalculatorFactory implements ICalculatorFactory {
    
    IConsoleFactory consoleFactory = new ConsoleFactory();
    
    IOperationFactory operationFactory = new OperationFactory();
    
    @Override
    public Operation createOperation(String type) {
        return operationFactory.createOperation(type);
    }
    
    @Override
    public AbstractConsole createConsole(String type) {
        type = (type == null ? "china" : type);
        return consoleFactory.createConsole(type);
    }
}

public class Main {
    
    public static void demo5() {
        ICalculatorFactory calculator = new CalculatorFactory();
        AbstractConsole console = calculator.createConsole(null);
        String lang = console.inputString("请输入语言");
        console = calculator.createConsole(lang);
        BigDecimal b1 = console.inputNumber();
        String opt = console.inputString();
        Operation operation = calculator.createOperation(opt);
        BigDecimal b2 = console.inputNumber();
        BigDecimal resp = operation.operation(b1, b2);
        console.output(resp);
    }
}
